# GitHub App Setup for Flux Authentication

This project uses a GitHub App for all Flux GitRepository authentication.
A single App identity works across configured repos. Prior to this change deploy SSH
keys were used which were generated by `flux bootstrap` each time this project was provisioned.

A GitHub App has a single persistent identity: the private key should be stored securely,
is injected into each cluster on bootstrap, and Flux's source-controller refreshes the short-lived
access tokens automatically.

---

## One-time setup

### 1. Create the GitHub App

1. Go to **GitHub → Settings → Developer settings → GitHub Apps → New GitHub App**
2. Fill in:
   - **Name**: `flux-playground` (or any name you like)
   - **Homepage URL**: `https://github.com/olga-mir/playground` (required but unused)
   - **Webhooks**: uncheck "Active" — Flux doesn't use incoming webhooks here
3. Set **Repository permissions**:
   - **Contents**: Read and write (needed for image-automation-controller to commit tag updates)
   - **Metadata**: Read-only (required)
4. Set **Where can this GitHub App be installed**: "Only on this account"
5. Click **Create GitHub App**
6. Note the **App ID** shown on the app settings page

### 2. Generate a private key

On the App settings page, scroll to **Private keys** → **Generate a private key**.
A `.pem` file downloads. Store the key material in a secret manager service

### 3. Install the App on your repos

1. On the App settings page, click **Install App** → select your account → **Only select repositories**
2. Select **both** `playground` and `playground-sre`
3. Click **Install**
4. After install, look at the URL: `https://github.com/settings/installations/XXXXXXXX`
   — that number is your **Installation ID**

## Configure local environment

Add these to `.setup-env` (never committed, already in `.gitignore`):

```bash
export GITHUB_APP_ID="..."
export GITHUB_APP_INSTALLATION_ID="..."
export GITHUB_APP_PRIVATE_KEY_FILE="wherever-it-is-when-fetched"
```

`GITHUB_FLUX_PLAYGROUND_PAT` is still used for the Flux notification provider (repository dispatch). A fine-grained PAT with **Contents: Read-only** is sufficient for that purpose.

---

## Configure GitHub Actions secrets

The `flux-bootstrap.yml` workflow bootstraps Flux on new GKE clusters. It needs:

| Secret name | Value |
|---|---|
| `GH_APP_ID` | App ID from the App settings page |
| `GH_APP_INSTALLATION_ID` | Installation ID from the install URL |
| `GH_APP_PRIVATE_KEY` | Full PEM content of the private key file |

Set them at: **playground repo → Settings → Secrets and variables → Actions → New repository secret**

---

## How it works at bootstrap time

1. `flux bootstrap github --token-auth` runs using the workflow's built-in `GITHUB_TOKEN` (which has `contents: write`). No App token needed — bootstrap only pushes files to the repo.
2. The script then replaces the `flux-system` K8s secret with the App credentials:
   `githubAppID`, `githubAppInstallationID`, `githubAppPrivateKey`
3. Flux source-controller picks up the App secret and handles all subsequent token refresh
   automatically — tokens are rotated before expiry with no manual intervention

The same `flux-system` secret is also used by the `playground-sre` GitRepository (which Flux
manages for the `sre` tenant), so installing the App on that repo covers it automatically.

## Verifying the setup

After bootstrap, confirm Flux is using App auth (no SSH keys should appear):

```bash
# Should show HTTPS URL, not ssh://
kubectl get gitrepository flux-system -n flux-system -o jsonpath='{.spec.url}'

# Secret should have App fields, not identity/identity.pub
kubectl get secret flux-system -n flux-system -o jsonpath='{.data}' | jq 'keys'
# Expected: ["githubAppID","githubAppInstallationID","githubAppPrivateKey"]
```
