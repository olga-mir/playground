# Upgrade Component Versions

This skill applies version updates from a pre-computed report generated by the bash pipeline.
The report is produced by the workflow step that runs before this skill is invoked.

## Prerequisites

`.version-report.md` must exist in the repo root, produced by:
```
bash bootstrap/scripts/scan-versions.sh | bash bootstrap/scripts/fetch-latest-versions.sh > .version-report.md
```

## Steps

### 1. Read the version report

Read `.version-report.md`. It contains a table with columns:
`Component | Type | File | Current | Latest | Update?`

### 2. Apply updates for each row marked **YES**

Use Edit to update each file. Skip these regardless of the report:
- `crossplane` component (requires special process, handled separately)
- `provider-gcp-beta-container` (to be replaced, skip for now)
- Any row where `latest_version` is `unknown`
- Do NOT read or modify `gotk-components.yaml`

For each file that needs updating:
- **type=helm**: update `spec.chart.spec.version: "OLD"` → `"NEW"`
- **type=crossplane**: update the version tag in `spec.package`, e.g. `:v0.5.0` → `:v0.6.1`
- **type=readme**: update the version link — both link text and URL contain the version string

If the same component version appears in multiple files (e.g. `providers.yaml` and
`providers-bootstrap/providers.yaml`), update ALL of them.

### 3. Also update README.md for YAML changes

For each helm or crossplane update, check if the component appears in the README.md
tech-stack table and update the version link there too (both link text and URL).

### 4. Create a PR if any files changed

After all edits:

```bash
DATE=$(date -u +%Y-%m-%d)
BRANCH="chore/upgrade-versions-${DATE}"
git checkout -b "${BRANCH}"
git add -A
git commit -m "chore: upgrade component versions"
git push origin "${BRANCH}"
gh pr create \
  --title "chore: upgrade component versions ${DATE}" \
  --body "..." \
  --base main
```

The PR body should include a summary table: component, old version → new version, files changed.

If nothing needed updating, just report that everything is up to date.

## Notes

- **LitmusChaos**: version from Helm index has no `v` prefix (e.g. `3.26.0`, not `v3.26.0`)
- **kagent / kgateway**: version from GitHub releases has `v` prefix — match the existing format in each file
- **FluxCD**: README-only update (no manifest to change)
- **Version format**: preserve the existing format in each file (with or without `v` prefix)
