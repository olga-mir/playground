Branch Refactor vs. main

A Visual Summary of Key Changes

FROM: Imperative Shell Scripts

  • Manual cluster deployment logic embedded in scripts (e.g., 03-deploy-gke-clusters.sh).
  • Complex, multi-step setup process that was hard to maintain and scale (04-setup-gke-clusters.sh).
  • High risk of configuration drift and manual errors.

TO: Declarative Crossplane

  • "Configuration as Code" using Crossplane Compositions to define GKE clusters (gke-cluster-composition.yaml).
  • New, simplified API for creating clusters (gke-cluster-xrd.yaml).
  • Infrastructure is now claimed via simple YAML files (apps-cluster-claim.yaml).
  • Automation is handled by a simple script that applies these configurations (03-apply-compositions.sh).
